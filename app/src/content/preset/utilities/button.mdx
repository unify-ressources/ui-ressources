### Introduction

The Button component in Preset UI offers a variety of customizable variants for colors and sizes, providing flexibility to match your design requirements.

## Base

Button comes with a base classes that set shared rules for all buttons.

Just add `btn` utility class.

<BlockCode>
```html
<button class="btn">
    Click Me
</button>
```
</BlockCode>

## Variants

Apply a solid, soft, ghost or Outline style to a button

| ClassName | Description | Example |
|-----------|-------------|---------|
| `btn-variantName` | Base class for all buttons with variantName style | `btn-solid` |
| `btn-variantName-colorName` | set colors for the button | `btn-solid-primary` |

## Size 

Set size for a button

| ClassName | Description | Example |
|-----------|-------------|---------|
| `btn-sizeVariant` | `xs`, `sm` ,`md`, `lg`,`xl` | `btn-md` |
| `btn-icon-sizeVariant` | `xs`, `sm` ,`md`, `lg`,`xl` | `btn-icon-lg` |



## Example

<BlockCode>
```html
<button class="btn btn-md btn-soft btn-soft-warning">
    Click Me
</button>
```
</BlockCode>


## Customization

All button variants and sizes are customizable. 
You can easily overwrite the default settings in your configuration to match your project's specific design needs.


| Title | Description | Type |
|-------|-------------|------|
| `ringBase` | Optional base properties for button rings | `RingBase` |
| `sizes` | Optional sizes available for buttons | `BtnSizes` |
| `iconSizes` | Optional sizes available for button icons | `BtnIconSizes` |
| `btnWhite` | Optional properties for `white` button variants | `BtnWhite` |
| `ring` | Optional properties for button rings with color shades | `RingColorShades` |
| `ringGray` | Optional properties for button rings with gray color shades | `RingColorShades` |
| `solidVariants` | Optional variants available for `solid` buttons | `BtnSolidVariants` |
| `outlineVariants` | Optional variants available for `outline` buttons | `BtnOutlineVariants` |
| `softVariants` | Optional variants available for `soft` buttons | `BtnSoftVariants` |
| `ghostVariants` | Optional variants available for `ghost` buttons | `BtnGhostVariants` |
| `gradientVariants` | Optional variants available for `gradient` buttons | `BtnGradientVariants` |

The `Button` type in Preset UI defines various optional properties that can be used to customize the appearance and behavior of the Button component. 
These properties include sizes for buttons (`sizes`), sizes for button icons (`iconSizes`), specific variants such as `solid`, `outline`, `soft`, `ghost`, and `gradient`, as well as configurations for button rings (`ringBase`, `ring`, `ringGray`), and `white` button variants (`btnWhite`).

## Example Config

Inside `presetUI` in your `uno.config` file :

<BoxCode title={"uno.config.ts"} icon={"unocss"}>
```js
presetUI({
	    components:{
            components:{
                sizes:{
                    //let's overide the default config for btn-xs
                    xs:{
                        height: 7,
                        px: 4.5,
                        textSize:"lg"
                    }
                },
                solidVariants: {
                    base: {
                        //for overiding default config for semantic button style like Primary, secondary,...
                        primary: {
                            bgShade: "500",
	                        hoverBgShade: "700",
	                        pressBgShade: "600",
                        },
                    },
                    global: {
                        // config for color that are not in base object or custom object
                        bgShade: "600",
	                    hoverBgShade: "900",
	                    pressBgShade: "700",
                    }
                },
            }
        },
    }
)
```
</BoxCode>
